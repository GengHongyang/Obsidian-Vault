### 2. 自定义异常类与日志

- **它是什么**：基于 `Exception` 创建自己的异常类型（如 `ConfigError`、`LLMAPIError`、`HistoryError`），在出错时抛出有意义的异常而不是用 `print` 或 `sys.exit`。同时用 `logging` 模块记录运行过程。
- **它解决 Agent 工程里的什么痛点**：Agent 运行时可能遇到配置缺失、API 超时、历史文件损坏等不同错误。没有分类异常会导致捕获时笼统处理，部署后难以排查。自定义异常让上层调用方可以按类型处理（如 LLMAPIError 自动重试，ConfigError 直接停止）。
- **今天为什么必须学**：今天是 Week 1 的倒数第二天，明天要写测试、复盘、整理面试话术。没有异常分类，测试无法精确断言错误类型，面试时也讲不出工程化细节。
- **新手最容易踩什么坑**：异常类只写 `pass` 没有任何额外属性；抛出异常后忘记在顶层捕获导致程序崩溃；把 `Exception` 当成万能捕获不分类。

```python
# app/exceptions.py
"""
自定义异常类，用于分类处理 Agent 运行中的配置、LLM 调用、历史记录等错误。
"""

class ConfigError(Exception):
    """配置相关错误：API Key 缺失、模型名错误等"""
    def __init__(self, message: str, status_code: int = 500):
        self.message = message
        self.status_code = status_code
        super().__init__(self.message)


class LLMAPIError(Exception):
    """大模型 API 调用错误：网络超时、认证失败、请求被拒绝等"""
    def __init__(self, message: str, original_exception: Exception | None = None):
        self.message = message
        self.original_exception = original_exception
        super().__init__(self.message)


class HistoryError(Exception):
    """历史记录读写错误：文件损坏、JSON 解析失败等"""
    def __init__(self, message: str, file_path: str = ""):
        self.message = message
        self.file_path = file_path
        super().__init__(self.message)
```

```python
# app/logger.py
"""
日志配置模块，统一日志格式和输出级别。
"""
import logging
import sys

def setup_logger(name: str = "llm_cli", level: int = logging.INFO) -> logging.Logger:
    """创建一个格式统一的日志记录器"""
    logger = logging.getLogger(name)
    logger.setLevel(level)

    # 避免重复添加 handler
    if not logger.handlers:
        handler = logging.StreamHandler(sys.stdout)
        formatter = logging.Formatter(
            "%(asctime)s - %(name)s - %(levelname)s - %(message)s",
            datefmt="%Y-%m-%d %H:%M:%S"
        )
        handler.setFormatter(formatter)
        logger.addHandler(handler)

    return logger

# 全局默认 logger
logger = setup_logger()
```
