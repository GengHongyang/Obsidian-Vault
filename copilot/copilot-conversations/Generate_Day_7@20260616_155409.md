### 1. pytest 基础与 Mock 测试思路

- **它是什么**：`pytest` 是 Python 最主流的测试框架。通过 `assert` 断言函数输出是否符合预期；Mock 测试则是在测试中模拟外部依赖（如 LLM API、文件读写），让你不用真实调用 API 也能验证代码行为。
- **它解决 Agent 工程里的什么痛点**：Agent 项目依赖外部 LLM API、数据库、文件系统。没有测试，每次修改都可能引入隐藏 Bug。有了 Mock，你可以离线验证核心逻辑，面试时也能说“项目有测试覆盖”。
- **今天为什么必须学**：Week 1 最后一天，必须把 LLM CLI 从“能跑”变成“可维护、可验证”。明天就是 Week 2 的 FastAPI 后端，没有测试习惯，后面会越写越不敢改代码。
- **新手最容易踩什么坑**：把测试写成了“print 大法”而不是用 `assert`；Mock 只 Mock 了函数名但忘了 Mock 返回值；测试文件命名没加 `test_` 导致 pytest 找不到。

```python
# tests/test_config.py
"""测试配置模块的读取和异常处理"""
import pytest
from unittest.mock import patch
from app.exceptions import ConfigError


def test_config_raises_error_when_key_missing():
    """当环境变量 LLM_API_KEY 缺失时，config.py 应抛出 ConfigError"""
    with patch.dict('os.environ', {}, clear=True):  # 清空环境变量
        with pytest.raises(ConfigError, match="LLM_API_KEY"):
            # 这里导入会触发 config.py 顶层的 ConfigError
            import importlib
            import app.config
            importlib.reload(app.config)  # 重新加载配置


# tests/test_history.py
"""测试历史记录模块的读写功能"""
import pytest
import json
import tempfile
from pathlib import Path
from app.history import save_history, load_history


def test_save_and_load_history(tmp_path: Path):
    """测试保存历史记录后能正确加载"""
    test_file = tmp_path / "history.json"
    test_data = [
        {"role": "user", "content": "你好"},
        {"role": "assistant", "content": "你好！有什么可以帮你的？"}
    ]

    # 保存
    save_history(test_data, file_path=str(test_file))
    assert test_file.exists(), "历史文件未被创建"

    # 加载
    loaded = load_history(file_path=str(test_file))
    assert loaded == test_data, f"加载的数据与保存不一致: {loaded}"


def test_load_empty_history_return_empty_list(tmp_path: Path):
    """当历史文件不存在时，应返回空列表而不是报错"""
    non_existent_file = tmp_path / "nonexistent.json"
    result = load_history(file_path=str(non_existent_file))
    assert result == [], f"应返回空列表，但得到 {result}"
```

```python
# tests/test_llm_client.py（Mock 测试示例）
"""测试 LLMClient 核心逻辑，Mock 掉真正的 API 调用"""
import pytest
from unittest.mock import AsyncMock, patch
from app.llm_client import LLMClient


@pytest.mark.asyncio
async def test_chat_returns_text():
    """Mock httpx 客户端，验证 chat 方法返回文本"""
    # 创建一个模拟的 httpx 响应
    mock_response = AsyncMock()
    mock_response.status_code = 200
    mock_response.json.return_value = {
        "choices": [{"message": {"content": "测试回复"}}]
    }

    # 模拟的异步上下文管理器
    mock_client = AsyncMock()
    mock_client.__aenter__.return_value.post.return_value = mock_response

    with patch('app.llm_client.httpx.AsyncClient', return_value=mock_client):
        client = LLMClient(api_key="test-key", base_url="http://fake.com")
        result = await client.chat("你好")
        assert result == "测试回复", f"期望得到'测试回复'，但得到 {result}"
```

### 你需要观察什么

1. 运行 `pytest -v` 后，每个测试是否显示 `PASSED` 或 `FAILED`。如果有 FAILED，阅读错误信息，理解是代码逻辑问题还是测试代码问题。
2. Mock 测试的重点：你并不需要真实的 API Key 也能测试 `LLMClient.chat()`，这就是 Mock 的价值。
3. `tmp_path` 是 pytest 内置的 fixture，每次测试自动创建临时目录，测试完成后自动清理——这是隔离测试文件的工程习惯。
