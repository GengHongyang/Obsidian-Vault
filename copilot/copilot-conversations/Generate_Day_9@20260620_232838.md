### 1. Pydantic 模型验证与 FastAPI 依赖注入

- **它是什么**：Pydantic 的 `Field` 可以对字段增加额外校验（长度、范围、正则、默认值、描述等），而 FastAPI 的 `Depends` 可以在路由处理函数之外提取和共享公共逻辑（如验证 Token、解析分页参数、注入配置对象）。两者组合可以实现一套声明式、可复用的 API 校验层。
- **它解决 Agent 工程里的什么痛点**：在 Agent 后端中，你会收到大量来自前端或用户的输入——对话消息、指令参数、历史记录 ID 等。手动用 `if` 检查每个字段既丑陋又容易遗漏。Pydantic + `Depends` 把校验逻辑集中到模型中，路由代码只关心业务，减少 50% 以上胶水代码。
- **今天为什么必须学**：昨天你写了第一个 FastAPI 路由和最简单的 schema。今天要深入——给 `POST /chat` 加上输入校验、可选字段、默认值，并用 `Depends` 注入配置对象，为后续注入 LLM Client、数据库会话等做准备。这是 Week 2 骨架中复用率最高的模式。
- **新手最容易踩什么坑**：在路由函数内部手动校验（如 `if not message.strip()`），而不是利用 Pydantic 的 `Field(min_length=1)`；忘记 `Depends` 只是函数调用，依赖项函数中也可以抛出 `HTTPException`；把 `query` 参数和 `body` 参数搞混导致请求 422。

关联知识：[[Pydantic 数据校验]]、[[FastAPI 路由]]、[[FastAPI 依赖注入]]

### 2. SQLite + Tortoise ORM 最小接入

- **它是什么**：Tortoise ORM 是一个 Python 异步 ORM，专为 FastAPI 等异步框架设计。SQLite 是零配置文件数据库，适合本地开发和原型验证。Tortoise 通过定义 `Model` 类映射到数据库表，支持 `create`、`filter`、`order_by` 等常见查询。
- **它解决 Agent 工程里的什么痛点**：Week 1 和 Day 8 的配置和数据存储在 JSON 文件中，没有索引、没有关联查询、没有并发安全性。一旦数据量变大（比如多个会话、用户、角色），JSON 文件读写会变成瓶颈。Tortoise ORM + SQLite 让你用几行代码就能完成数据建模、插入和查询，且异步不阻塞。
- **今天为什么必须学**：今天你要为 `week2_agent_backend` 增加数据库持久化。上午完成 Tortoise 初始化配置，下午写 `Conversation` 和 `Message` 两个模型，并修改 `POST /chat` 路由将用户消息和 AI 回复写入数据库。这是后续所有后端功能的基础。
- **新手最容易踩什么坑**：安装 `tortoise-orm` 后忘记安装 `aiosqlite`；在 `app/main.py` 中调用 `Tortoise.init(...)` 的位置不对（应该在 startup 事件中），导致第一次请求时表还未创建；字段类型选择错误，如用 `CharField` 存储长文本应该用 `TextField`。

关联知识：[[Tortoise ORM 数据建模]]、[[SQLite 数据库]]、[[FastAPI 项目结构]]

```python
"""对话模型：存储一次对话的基本信息"""
from tortoise.models import Model
from tortoise import fields


class Conversation(Model):
    """一次对话（会话）的数据库模型"""
    id = fields.UUIDField(pk=True)          # 主键，自动生成 UUID
    title = fields.CharField(max_length=200, default="New Chat")  # 对话标题
    created_at = fields.DatetimeField(auto_now_add=True)          # 创建时间

    class Meta:
        table = "conversations"             # 数据库表名

    def __str__(self) -> str:
        return f"{self.title} ({self.id})"
```

```python
# app/models/message.py
"""消息模型：关联到某一对话"""
from tortoise.models import Model
from tortoise import fields


class Message(Model):
    """单条消息的数据库模型，关联到一个 Conversation"""
    id = fields.IntField(pk=True)                            # 自增主键
    conversation = fields.ForeignKeyField(
        "models.Conversation", related_name="messages"       # 外键：多条消息属于一个对话
    )
    role = fields.CharField(max_length=10)                   # "user" 或 "assistant"
    content = fields.TextField()                             # 消息内容（长文本）
    created_at = fields.DatetimeField(auto_now_add=True)     # 发送时间

    class Meta:
        table = "messages"

    def __str__(self) -> str:
        return f"[{self.role}] {self.content[:50]}..."
```

```python
"""聊天接口的请求/响应 schema，使用 Pydantic Field 进行校验"""
from pydantic import BaseModel, Field
from uuid import UUID
from typing import Optional


class ChatRequest(BaseModel):
    """聊天请求体"""
    message: str = Field(
        ...,
        min_length=1,
        max_length=2000,
        description="用户输入的文本，长度 1-2000 字符",
    )
    conversation_id: Optional[str] = Field(
        default=None,
        description="对话 ID（UUID 字符串）。不传则创建新对话",
    )


class ChatResponse(BaseModel):
    """聊天响应体"""
    reply: str = Field(..., description="AI 回复内容")
    conversation_id: str = Field(..., description="当前对话 ID")
    model: str = Field(..., description="使用的模型名称")
```

```python
# app/routers/chat.py
"""聊天相关路由"""
from fastapi import APIRouter, Depends, HTTPException
from uuid import uuid4, UUID
from app.schemas.chat import ChatRequest, ChatResponse
from app.models.conversation import Conversation
from app.models.message import Message
from app.dependencies import get_settings
from app.config import Settings

router = APIRouter(prefix="/chat", tags=["chat"])


@router.post("", response_model=ChatResponse)
async def create_chat(
    request: ChatRequest,
    settings: Settings = Depends(get_settings),  # 注入配置对象
) -> ChatResponse:
    """处理用户消息：持久化到数据库并返回假回复（明天替换为 LLM 调用）"""
    # 1. 获取或创建 Conversation
    if request.conversation_id:
        try:
            conv_id = UUID(request.conversation_id)
            conversation = await Conversation.get_or_none(id=conv_id)
            if conversation is None:
                raise HTTPException(status_code=404, detail="对话未找到")
        except ValueError:
            raise HTTPException(status_code=400, detail="无效的对话 ID 格式")
    else:
        # 创建新对话，标题取消息前 20 个字符
        title = request.message[:20] + "..." if len(request.message) > 20 else request.message
        conversation = await Conversation.create(title=title)

    # 2. 保存用户消息
    await Message.create(
        conversation=conversation,
        role="user",
        content=request.message,
    )

    # 3. 生成假回复（占位，Day 10 替换为真实 LLM 调用）
    fake_reply = f"你说了：{request.message[:30]}...（这是占位回复，明天接入真实模型）"

    # 4. 保存 AI 回复
    await Message.create(
        conversation=conversation,
        role="assistant",
        content=fake_reply,
    )

    # 5. 返回响应
    return ChatResponse(
        reply=fake_reply,
        conversation_id=str(conversation.id),
        model=settings.llm_model,   # 从注入的 settings 获取
    )
```
