📌 **今日核心死磕**

### 1. FastAPI 项目结构（[[FastAPI 项目结构]]）
- **它是什么**：FastAPI 是 Python 异步 Web 框架，自动生成 OpenAPI 文档，天然支持 `async/await`。合理的项目结构包括 `main.py`、`routers/`、`schemas/`、`services/`、`core/` 等分层。
- **它解决 Agent 工程里的什么痛点**：Agent 项目需要暴露 HTTP 接口给前端或其他服务调用。没有后端骨架，LLM Client 只能跑在命令行，无法集成到面试系统等实际业务中。
- **今天为什么必须学**：Week 2 的核心产出物是“Agent 后端服务骨架”，今天打好结构，后续每天往里加模块（认证、ORM、RAG）。
- **新手最容易踩什么坑**：把所有逻辑塞进 `main.py`，导致单文件臃肿；不理解 `router` 和 `app.include_router` 的分离意义。

### 2. Pydantic 请求/响应模型（[[Pydantic 数据校验]]）
- **它是什么**：Pydantic 模型用类型注解定义数据结构，自动验证输入、序列化输出，FastAPI 依赖它生成 API 文档。
- **它解决 Agent 工程里的什么痛点**：LLM API 的 messages 结构、用户输入、响应格式都需要严格校验。Pydantic 保证数据在进入业务逻辑前就是干净的，否则返回 422 错误。
- **今天为什么必须学**：第一次基于 FastAPI 写端点，必须掌握 `BaseModel`、`Field`、`validator`，否则后续 CRUD 会反复踩坑。
- **新手最容易踩什么坑**：忘记继承 `BaseModel`；字段类型写错（比如 `list` 没有指定内部类型）；不在模型层面校验空值或长度。

```python
# app/schemas/chat.py
"""定义 Chat API 的请求和响应 Pydantic 模型"""
from pydantic import BaseModel, Field
from typing import Optional


class Message(BaseModel):
    role: str                                  # "user" | "assistant" | "system"
    content: str


class ChatRequest(BaseModel):
    messages: list[Message]                    # 至少包含一条消息
    temperature: float = Field(default=0.7, ge=0, le=2)


class ChatResponse(BaseModel):
    reply: str
    usage: Optional[dict] = None
```

```python
# app/services/llm_client.py
"""从 Week 1 搬过来的 LLMClient，使用全局 settings"""
import httpx
from app.core.config import settings
from app.exceptions import LLMAPIError          # 从 week1 项目复制，或者新建


class LLMClient:
    def __init__(self):
        self.api_key = settings.llm_api_key
        self.base_url = settings.llm_base_url
        self.model = settings.llm_model

    async def chat(self, messages: list[dict], temperature: float = 0.7) -> dict:
        """异步调用 LLM API，返回原始响应字典"""
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json",
        }
        payload = {
            "model": self.model,
            "messages": messages,
            "temperature": temperature,
        }
        async with httpx.AsyncClient(timeout=30) as client:
            try:
                resp = await client.post(
                    f"{self.base_url}/chat/completions",
                    headers=headers,
                    json=payload,
                )
                resp.raise_for_status()
                return resp.json()
            except httpx.HTTPStatusError as e:
                raise LLMAPIError(f"API 返回错误状态码: {e.response.status_code}")
            except httpx.TimeoutException:
                raise LLMAPIError("API 请求超时")
```

```python
# app/routers/chat.py
"""Chat API 路由"""
from fastapi import APIRouter, Depends, HTTPException
from app.schemas.chat import ChatRequest, ChatResponse
from app.services.llm_client import LLMClient
from app.exceptions import LLMAPIError

router = APIRouter(prefix="/api/v1/chat", tags=["Chat"])


# 依赖项：从全局单例获取 LLMClient
async def get_llm_client() -> LLMClient:
    # 实际项目中可以用更优雅的单例管理，这里简单示意
    from app.main import app
    if not hasattr(app.state, "llm_client"):
        app.state.llm_client = LLMClient()
    return app.state.llm_client


@router.post("", response_model=ChatResponse)
async def chat(request: ChatRequest, llm: LLMClient = Depends(get_llm_client)):
    """发送消息给 LLM，返回回复"""
    # 将 Pydantic Message 列表转为普通字典列表，因为 LLMClient 需要 dict
    messages_dict = [msg.model_dump() for msg in request.messages]
    try:
        result = await llm.chat(messages_dict, temperature=request.temperature)
        reply = result["choices"][0]["message"]["content"]
        usage = result.get("usage")
        return ChatResponse(reply=reply, usage=usage)
    except LLMAPIError as e:
        raise HTTPException(status_code=502, detail=str(e))
```
